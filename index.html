流程步骤及规则介绍

审单-订单克重筛查-最新版
已完成
程序版本： 版本1 发布时间：2025.10.18
技术栈： 影刀RPA + Decimal + HTTP
执行频率： 每天执行两次
项目进度 100%
定时任务配置：
每天 3时00分 执行流程 审单-订单克重筛查-正式 审单第二批数据(18：01-23：59) 推送至钉钉审单复核群
每天 18时00分 执行流程 审单-订单克重筛查-正式 审单第一批数据(0：00-18：00) 推送至钉钉审单复核群
任务流程介绍：
第一步 下载指定时间段的 符合条件的订单数据
第二步 筛选并剔除非金订单 即银饰等
第三步 根据内部订单号请求聚水潭并获得发货的克重
第四步 获取聚水潭线上规格克重 允许克重上下浮动 订单克重可以高于商品资料克重0.2g 同理可以低于0.02g
第六步 将订单克重发货错误的信息数据生成表格并推送至钉钉群聊 审单复核群
数据-统计件数及总重
已完成
程序版本： 版本5 发布时间：2025.10.26
技术栈： 影刀RPA + Decimal + HTTP
执行频率： 每天执行一次
项目进度 100%
定时任务配置：
每天 9时00分 执行流程 数据-统计件数及总重 统计进货仓足金调货克重件数提醒 推送至钉钉审单复核群
任务流程介绍：
第一步 请求进货仓所有 状态已完成 排除已取消的调拨单
第二步 根据调拨单号查询此单号的唯一码明细并获取克重总重
第三步 为确保准确性采用国际金额计算方式 严格确保克重的精准性避免偏差
第四步 根据需求可选择生成表格明细提供审查准确性
第五步 通知信息件数和总重到钉钉群质检部信息通知
数据-宇航条码筛查
已完成
程序版本： 版本23 发布时间：2025.10.27
技术栈： 影刀RPA + PyThon + HTTP
执行频率： 每天执行一次
项目进度 100%
定时任务配置：
每天 10时00分 执行流程 数据-宇航条码筛查 筛选近15天店铺条码绑定出错的数据
任务流程介绍：
第一步 下载指定时间段的（十五天内） 符合条件的店铺商品信息 数据
第二步 筛选并剔除得物店铺和已关闭店铺数据，并剔除银饰和18k金以及唯品会数据
第三步 剔除指定商品编码如 lingfengjin，1234，1235，123等
第四步 提取线上克重信息并对比请求聚水潭返回的普通商品资料信息
第五步 判断克重误差区间是否在0.1-0.03之间，异常数据则分店铺分表格保存记录
第六步 将异常数据表格按照店铺推送至对应店长的钉钉（注 小豪的推送到中台群聊）
